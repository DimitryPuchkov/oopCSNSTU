 Инструкция по использованию git:
  

Начало работы:
	а) нет репозитория на компьютере:
		git clone https://github.com/DimitryPuchkov/oopCSNSTU.git
	b) продолжаешь работу с репозиторием:
		git pull

После после того, как поработала и хочешь сохранить свои измения:
	git add -A (git add --all; git add ./   - возможные варианты)
	git commit -m"[тут пишешь о том, что сделала]" (не забудь к команде дописать опцию -m, иначе попадешь в вим, а выйти из него не так просто
	

Для отправки на удаленный репозиторий своей работы:
	git push



***********************************************Справочник**************************************************


git clone [url]   - скопировать репозиторий в данную папку
			(не пользуйся этой командой для загрузки последней версии репозитория, если 
			 необновленная версия этого репозитория уже есть, для обновления существует команда pull)



git pull         -  обновить локальную версию репозитория, возможно появление окошка vim, в этом случае нужно нажать Esc :wq Enter
                    !!! Бывают случаи когда заранее известно что при выполнениии этой команды могут возникать конфликтные ситуации
                        (большие различия в коде который сейчас локально и который приходит из удаленного репозитория)
                        в этом случае, если нужно принять все изменения и разрешить конфликты в пользу принимаемых изменений, то 
                        использовать следующюю команду:
                        git pull -X theirs


git add ./       - зафиксировать изменения


git commit -m"..."  - сделать коммит изменений (окончательно зафиксировать изменения и написать комментарий)


git push         - отправить изменения на гитхаб
